Bug监管方案

1. 前期准备

  • 参予组长周会，记录每周提测项目及负责人;
  • 建立每周负责人讨论组，方便沟通;

2. 监管实施

  • 每天下午18点，各项目负责人提供如下信息（具体到每个项目参与者）格式如下：
    参与人A：
        ·0 昨日未完成数
        ·1 今日新增数
        ·2 已解决数
        ·3 未解决数
        ·4 未解决原因
        ·5 预计解决时间
    参与人B：
        ·0 昨日未完成数
        ·1 今日新增数
        ·2 已解决数
        ·3 未解决数
        ·4 未解决原因
        ·5 预计解决时间

3. 统计上报

  • 汇总各项目bug情况；
  • 发送至负责人讨论组由负责人核验，无误后发送至QQ群；

4. 监管标准

  • 考虑初测试阶段bug量较多，进入测试平台期后（bug量减少情况基本稳定），为鼓励和推动bug修复工作，依照如下标准每周进行评比：

  • 评比细则：（落实到每个项目参与者）
    ·0 时间标准：当日下午三点之前的bug,当日完成，不能完成需提供原因及预计完成时间；
    ·1 数量标准：当日bug数量如少于十个需要当天完成修改任务，超过十个且不能完成需提供原因及预计完成时间；
    .3 每周逾期次数大于3次，团队贡献费10元/人。
